# v0.6.0

## Highlights

- Expanded editable report-panel layout controls on `BoxStyle(...)`:
  - explicit `width` and `unit`
  - per-side padding with `padding_top`, `padding_right`, `padding_bottom`, and `padding_left`
  - per-box `alignment`
  - `title_text_color`
- Made nested tables and figures render more consistently inside boxes across DOCX, PDF, and HTML.
- Added DOCX rendering for the shared fixed-sheet shape subset:
  - `Shape.rect(...)`
  - `Shape.ellipse(...)`
  - `Shape.line(...)`
- Updated the usage guide to prefer editable `Box(...)`, `Table(...)`, and `Figure(...)` report panels for tcolorbox-like layouts.
- Added a LaTeX-to-docscriptor transition section to the usage guide so authors can map familiar commands such as sections, figures, labels, tables, citations, and tcolorbox-style panels to the Python-first API.
- Aligned the package fallback version metadata with the v0.6.0 release line.
- Added `Document.save(path)`, which chooses DOCX, PDF, or HTML rendering from the output file extension.
- Added `Document.save_all(output_dir)`, which writes the normal DOCX/PDF/HTML review bundle in one call.
- Added Word-style inline authoring helpers for highlighted text, strikethrough text, and manual line breaks inside a paragraph.

## Notes

- `Sheet(...)` remains available for fixed-layout form inserts, but report-style layouts should prefer editable flow blocks when Word editability matters.
