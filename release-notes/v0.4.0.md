# docscriptor v0.4.0

## Highlights

- Added page-level layout controls:
  - `PageSize` and `PageMargins` can be supplied through `DocumentSettings`.
  - DOCX, PDF, and HTML now share the same page size and margin intent.
  - `DocumentSettings` and `Document` expose helpers such as `get_text_width(...)` and `get_page_width(...)`.
- Expanded length handling:
  - Document-level `unit=...` controls numeric dimensions.
  - Supported units include inches, centimeters, millimeters, points, pixels, picas, and twips.
- Improved figure sizing:
  - `Figure(width=...)`, `Figure(height=...)`, or both dimensions can be used.
  - Single-dimension sizing preserves aspect ratio; two-dimensional sizing uses the explicit box.
  - Figure sizes can be derived from page/text geometry with helpers such as `settings.get_text_width(0.75)`.
- Added explicit pagination with `PageBreak()`.
- Added document page background color with `Theme(page_background_color=...)`.
- Reworked generated contents pages:
  - TOC entries show dotted leaders and page labels by default.
  - `TableOfContents(...)` can hide page labels, limit depth, change leaders, and customize per-level styles with `TocLevelStyle`.
  - Top-level TOC entries are emphasized by default while lower levels use normal weight and tighter spacing.
- Updated the usage guide so the new page layout, figure sizing, and TOC controls are documented with examples.

## Breaking Changes

- Removed the older `Figure(width_inches=...)` API. Use `Figure(width=..., unit="in")` for explicit inches, or rely on `DocumentSettings(unit=...)` for document-wide units.

## Notes

- This is a minor release because backward compatibility is not guaranteed.
- The repository URL now follows the renamed GitHub origin: `https://github.com/Gonie-Gonie/docscriptor`.
